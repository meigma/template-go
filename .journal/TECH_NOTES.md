# Technical Notes

- The template intentionally exercises releases with its example app before generated projects inherit the setup. The nominal path is binary plus container, but generated projects may be binary-only, container-only, or pure libraries; `DELETE_ME.md` documents what to trim for each shape.
- Tool versions + integrity are owned by **mise** (`mise.toml` + `mise.lock`, `[settings] locked = true` → fail closed), which replaced Proto (`.prototools`/`.moon/proto/*`). moon runs on the `system` toolchain (bare binaries on PATH, `toolchains.default: system`); CI installs via `jdx/mise-action`, not `moonrepo/setup-toolchain`. Bump = edit `mise.toml`, `mise lock --platform linux-x64,linux-arm64,macos-x64,macos-arm64`, commit both. There is no `.go-version`; the Go version lives in `mise.lock` + `go.mod` (`GOTOOLCHAIN=local`). See the `mise`/`melange`/`apko` skills under `.agents/skills`.
- GoReleaser builds the binary assets. The container image is built **without a Dockerfile**: melange (source → signed Wolfi apk, `melange.yaml`) + apko (apk + Wolfi base → minimal multi-arch nonroot OCI image, `apko.yaml`), published to GHCR, signed with keyless cosign, with a syft image SBOM. Native amd64/arm64 hosted-runner builds, no QEMU. `mise run image-local` builds it locally.
- Provenance for both binaries and the image is generated in an **isolated reusable workflow** `.github/workflows/attest.yml` (its own OIDC identity → SLSA Build L3) while staying on GitHub's attestation API. The signer-workflow is `attest.yml` — set in `ghd.toml` `signer_workflow`, `stage_ghd_release_assets.py` `expected_signer`, and the dry-run check; verify with `gh attestation verify --signer-workflow .../attest.yml`. The `attest-binaries` caller must grant `packages: write` (the job is shared with `attest-image`), and `attest.yml` does its own GHCR login before the image-referrer push. cosign signing + the syft SBOM attestation stay in the build job (only provenance is isolated).
- The publishing pipeline (`release.yml`) sets the `jdx/mise-action` `cache: false` so a poisoned Actions cache cannot influence the toolchain that builds/signs real artifacts (Kusari-flagged hardening). CI/dry-run/security-scan keep the cache — they publish nothing.
- The scheduled security scan workflow is drift detection, not a required PR gate. It builds the local amd64 image via melange + apko weekly/manual-only, scans high/critical fixed vulnerabilities with Trivy, and uploads SARIF to GitHub code scanning.
- Release Please must use bare `vX.Y.Z` tags here: keep `include-component-in-tag: false` and `force-tag-creation: true` in `release-please-config.json` (force-tag-creation is what creates the git tag — a draft release alone does not). `melange.yaml`/`apko.yaml` carry `# x-release-please-version` markers and are in `extra-files`.
- The tag-triggered release workflow resolves a pre-existing draft release without checkout. Keep `contents: write` on `resolve-release` and pass `--repo "$GITHUB_REPOSITORY"` to `gh release view`.
- Required default-branch checks are `ci`, `Binary Release Dry Run`, and `Container Image Dry Run` (job names preserved across the melange/apko migration).
- CI uses GitHub-native caches for Go modules, Go build artifacts, golangci-lint, and uv downloads, keyed off `mise.lock`. There is no Docker/BuildKit image build (melange/apko replaced it). Treat Moon remote cache as a later opt-in requiring a Bazel Remote Execution-compatible backend and credentials.
- GitHub-specific helper scripts live under `.github/scripts`. The release workflow uses `stage_ghd_release_assets.py` to validate `ghd.toml` (including that `signer_workflow` is `attest.yml`), stage GoReleaser binary/SBOM/checksum assets, and verify release checksums before upload.
- Docs use MkDocs plus `uv` (`docs/mkdocs.yml`, `docs/uv.lock`), not Docusaurus/npm. Obsolete Dependabot PRs or stale security-update jobs for `docs/package.json` / `docs/package-lock.json` should be closed or dismissed rather than reintroducing those files.
- `.agents/skills` holds committed agent skills (`git`, `worktrunk`, `session-*`, plus the `mise`/`melange`/`apko` tooling skills). `.gitignore` blanket-ignores `.agents/`, so committed skills are force-added (`git add -f`); the downstream `template-go-api` refined its `.gitignore` to track `.agents/skills` normally — a pending template improvement.
